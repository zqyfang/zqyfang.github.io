Java查漏补缺（三）

发表于 2017-08-06 | 分类于 Java

Java开始

书接上文，继续开始啦！^_+

第五部分

  • 多态：
    A：要有继承关系
    B:要有方法重写
    C：要有父类引用指向子类对象
输出：30show，name，40showzi1
解析：因为多态时候，方法存在重写，但是变量不存在重写，
所以对于成员方法，编译看左边，运行看右边，对于成员变量，都看右边。
  • 抽象
    抽象类，抽象方法都被abstract修饰，抽象方法没有方法体（{}内部的部分）。
    A:有抽象方法的类一定是抽象类或接口
    B：抽象类中不一定有抽象方法。
    C:抽象类不能被实例化。
    构造方法作用：实例化对象，并且初始化成员变量。但是抽象类时候类都是抽象的了，更不可能实例化出对象，所以构造方法只是为了初始化成员变量
    抽象方法必须没有方法体，而且必须abstract修饰。
    public abstract void name();
  • 接口
    interface Animal{}
    接口和抽象类区别：接口全是抽象方法（当然也可以啥也没有），比抽象类更抽象（但是也可以放常量）。
    A:接口不能被实例化
    B:接口中没有变量，只有常量，因为接口中变量都有默认修饰符，public static final。
    C:接口没有构造方法（抽象方法有，因为抽象方法需要初始化变量），但是接口只有静态常量，随类加载而加载。
    D:接口中方法都是抽象的。即使不加abstract也是抽象的，默认有public abstract。
    E:类实现接口时候，可以多实现（不是多重实现），并且可以在继承一个类基础上实现多个接口
    F：接口和接口可以继承，可以多继承。

  • 接口和抽象类区别：
    抽象类：共性的时候使用 is a.
    接口：扩展的时候，like a.

    例如品乓球运动员，篮球运动员，乒乓球教练，篮球教练
    可以抽象为运动员，教练。
    这两个可以抽象为人。
    但是会说英语，例如乒乓球运动员和篮球教练会说英语，就是使用接口。
    教练 is 人，但是英语 is 人几部合适了。

    第六部分

  • ==比较符
    基本类型时候，比较的是值是否相同
    引用类型时候，比较的是地址值
  • 方法被调用，在方法内部就有一个this代表该对象。
    这样可以简化参数传递，只传入一个即可。
  • equal：针对上面的改进版以及解析
    object中有equal的方法

但是此时默认的比较是比较两个对象地址值，但是地址值无效，所以此操作也是没有意义的。故而，需要重写此方法。

  • scanner相关：

注意：可以输入多次，根据接收几次来确定,但是先int,后string不行，因为会把enter读取string。解决方案：在new一个scanner，或者字符串拼接。

第七部分

  • 冒泡排序。
  • 折半查找也叫二分法查找（前提是数组有序）
  • String和StringBuffer类：
    string一旦赋值不能改变。
    咱们一般意义的改变，其实只是字符串对应的常量池地址变化而已
    案例：
    String a=”hha”;
    String b=”hhe”;
    String c=a+b;
貌似可以赋值，其实是a在方法区的常量池有个地址，b有另一个地址，但是c被赋值为a+b时候，
相当于把a和b相连接，但是在内存中新开辟一个内存地址进行赋值，所以，字符串值不能改变，
但是引用可以改变，照成了貌似可以改变值的假象。
也就是说c==a+b返回false。

但是StringBuffer值可以改变，其是字符串缓冲区的意思：

相关方法：
末尾添加：append(),返回一个StringBuffer，而且修改，修改的都是本身。
指定位置添加:insert(8,"world"),而且修改，修改的都是本身。
delete(int index):删除指定位置字符
deleteCharAt(int start,int end):删除strat到end位置字符，包左不包右。

  • 题目一、

但是String类比较特殊，不会直接改变

  • 题目二：如果替换成StringBuffer，则两次输出都是哈哈黑恶虎。
  • 题目三、

c==”哈哈”+”哈哈1”赋值是常量赋值，基本数据类型中，常量如果存在不会重新创建所以c和”哈哈”+”哈哈1”都指向”哈哈哈哈1”。但是c==a+b时候，是变量赋值，所以都会在内存中新建。最终指向不同地址。

  • Integer类
Interger.parseInter().
JDK1.5新特性：
Integer i=new Integer(100);
Integer i=100;//等效，默认相当于new Integer(100)//自动装箱
i=i+200;//按理说会报错，int+Integer不行，但是底层有自动拆箱，相当于i.intValue().
把上面转换成i.intValue()+200.
i=i.intValue()+200；计算结果为300，但是i为Integer，所以又一次自动装箱，转换为Integer类型。
