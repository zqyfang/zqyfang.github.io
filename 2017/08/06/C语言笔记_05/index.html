C语言笔记_05(结构体相关)

结构体基本概念

struct Person p;//定义结构体变量
定义同时初始化:struct Dog={“wc”,12,5.0};
先定义之后，然后一次性初始化：
struct Dog d1;
d1=(struct Dog){“xq”,6,7.0};类似于强制类型装换
还可以通过结构体名，逐个初始化例如：d1.name=”xc”；
将值赋值给指定属性：
struct Dog d1={.height=1.73，.name=”wc”};

注意：
结构体内部属性内存寻址是从小到大
结构体名称并不是结构体地址
结构体地址第0个属性的地址
结构体占用内存空间：查找所有属性中占用内存空间最大的属性，然后按照
属性的倍数(比如说一倍，两倍等，系统自动设置倍数)分配内存空间，类似
于比如有8个字节的属性，先创建一个8字节的内存空间，用完之后，在开辟
一个，如果最后比如说剩下四个字节，但是只有一个char(一个字节)需要存
储的时候，则不在开辟了，多余空间就不需要管了。
定义结构体变量的时候才会分配内存空间

类型定义方式

结构体作用域

如果结构体定义在函数或者代码块内部，则结构体作用域和变量相同。

指向结构体的指针

定义指向结构体的指针：
   struct Person{
        int age;
        char *name;
        double height;
    };
    struct Person p={30,"lnj",1.75};
    //定义指针
    struct Person *pp;
    pp=&p;
    (*pp).age=40;//必须加括号，因为。运算符的优先级高于*
    pp->height=20.0;//还可以通过这种方式赋值修改值
    printf("%d\n",(*pp).age);
    printf("%lf\n",(*pp).height);//lf:针对的是double类型的占位符
    printf("%f\n",pp->height);

结构体和函数

结构体的属性传递给函数，在函数中修改形参不会影响到实参
结构体名称作为参数传递，函数中修改形参不会影响到实参，结构体之间是值传递。
例如把结构体p1赋值给p2，则p1中属性修改不会影响到p2属性值
案例：函数中参数是结构体的写法
void change(struct Person p){};//不会修改属性值
void change(struct Person *p){};//指针，会属性修改值
