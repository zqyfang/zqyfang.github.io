C语言笔记_02

函数

注意点

在函数中修改形参的值，不会影响到外面实参的值(前提是：基本数据类型)
返回值和返回值类型不同的时候，会进行隐式转换

int getHeight(){
    return 1.75;
}
如果函数没有返回值，返回值类型是void
void setHe(){
    printf("案发后沙");
}
C语言可以没有返回值类型：没有写，默认为int类型。
test(){
    return 1;
}

函数的声明和实现

C语言中默认在调用之前，声明定义函数，如果不定义函数，一定要在调用之前声明函数。
#include <stdio.h>
void showHello();//函数声明
int main() {
    showHello();
    return 0;
}
void showHello(){//函数定义
    printf("hello");
}
注意：如果函数返回值是int类型，那么不写声明，也不会报错。而且函数声明可以省略形参，
函数声明只需要在函数被使用之前就可以了，它可以写在函数的外面，也可以写在函数里面。
C语言中函数定义不能重复，但是函数声明可以重复，因为函数声明仅仅是为了告诉系统有这个函数，
接受什么，返回什么，所以多告诉几次也无所谓。

main函数解析

main系统规定，程序入口，只能由系统调用，不能手动调用
系统启动时候，会给程序传递进来一些参数
argc:系统传递进来的参数个数
argv:系统传递进来参数的实际值
int main(int argc,const char *argv[]) {
    printf("%i\n",argc);
    printf("%s\n",argv[0]);
    return 0;//返回给操作系统，代表正常结束，如果返回不是0，代表非正常结束
}

mac常用指令

pwd:查看当前位置
ls:查看当前文件夹中的内容
cd Desktop/:进入指定文件夹，此时进入Desktop文件夹
mkdir:新建文件夹，指令后面跟上文件夹名称（mkdir qiang）
rmdir: 文件夹名称/:删除文件夹
cd ../:返回上一个文件夹（注意空格）
clear:清空终端
open 文件名称/:打开文件
touch 文件全称:创建文件
cat 文件全称:预览文件

include指令(预处理指令)

不一定要写在文件头部，作用就是把对应文件copy进来，替换掉对应函数

“”和<>区别：<>代表先从开发工具的编译环境中找寻文件，没有的话再去系统的编译环境中
查找, “”代表会先从当前文件夹下面查找，没找到则去开发工具的编译环境中查找，如果还
是没找到去系统的编译环境中查找。
“”：不一定非要写文件名，还可以写相对路径，或者绝对路径

进制

C语言中数字前面加0b代表二进制，0代表八进制，0x表示十六进制
案例：八进制转化为十进制输出
int num=12;
printf(“八进制:%o\n”,num);//14，o代表八进制
printf(“十六进制:%x\n”,num);//c，x代表十六进制
另外 i代表十进制
注意：C语言中没有以二进制输出的占位符。

原码，补码，反码

数据存储中内存都是存储的二进制，最终存储在内存中的是补码
反码：正数的反码就是正数原码
补码：正数的补码就是正数的原码
所以：正数，三码合一。
案例：通过-9说明负数的存储情况
 -9是int类型，在mac系统上面，占据四个字节，也就是32位。
二进制(原码)第一位是符号位，1代表负数，0代表正数。
 -9的原码就是：10000000 00000000 00000000 00001001
反码为符号位不变，其他位取反
 -9的反码就是：11111111 11111111 11111111 11110110
补码为反码加一
 -9的补码就是：11111111 11111111 11111111 11110111 
总结：之所以有原码，补码，反码的区分，是因为为了方便计算机计算。因为原码最
高位是符号位，如果直接用原码计算，还需要判断是整数还是负数，但是如果用反码，
补码此时已经不区分符号位了，直接计算就行，提高了效率。
案例：1+（-1）=0
00000000 00000000 00000000 00000001  1的补码
11111111 11111111 11111111 11111111  -1的补码
——————————————————————————————————————
00000000 00000000 000000000 00000000 
解析：本来最高位应该是1，但是因为超出四字节，被截取，此时最高位为0为正数，顾结果不需要考虑三码，直接为0.
案例二：-9加1
11111111 11111111 11111111 11110111  -9补码
00000000 00000000 00000000 00000001   1补码
——————————————————————————————————————
11111111 11111111 11111111 11111000   -8补码
解析：最高位为1是负数，先求反码为：11111111 11111111 11111111 11110111
然后求原码为：10000000 0000000 00000000 00001000 此时为-8.

位运算

按位与 & 一假则假 0代表假，1代表真

案例：9&5
 00001001
&00000101
——————————
 00000001  结果为1

按位或 | 一真则真

 00001001
|00000101
——————————
 00001101  结果为13

按位亦或 ^ 不同为1，相同为0

 00001001
|00000101
——————————
 00001100 结果为12
说明：亦或结果和参与运算顺序无关，9^5^6=6^5^9.
相同两个数亦或为0，任何数亦或0还是本身，任何一个数和同一个数亦或两次还是本身。

按位取反 ~

~00001001=11110110(补码)
11110101（反码）
10001010 （原码） 结果为-10

移位运算

左移：<<  左移有可能改变数的正负性，因为左移最高位（符号位）会被抛弃，后面0。
9<<1=9*2(1)=18
右移: >> 右边移动位被抛弃，左边缺失位补充符号位。
9>>1=9/2(1)=4 C语言中整数除以整数等于整数
注意：负数移位(左右)的时候，需要转换为补码在进行移位，移位后，在转化为源码得到值。

案例：交换值
方式一：
int temp=a;
a=b;
b=temp;
方式二：
a=a+b;
b=a-b;
a=a-b;
方式三：
a=a^b;
b=a^b;
a=a^b;

char类型

char占据一个字节
可以直接把整型赋值给一个char。
char a=6;
printf("%i\n",a);//6
注意：所以在不超过char取值范围的情况下，char(-127~128)可以当做int使用

类型说明符号

说明长度的（可用于修改类型所占用的存储空间的大小，一般和int配合使用）

short: short int为两个字节，short int=short。输出使用%hi或者%hd。
long:例如long int num=12411;本来int是四个字节，但是加上之后，为8个字节。long int=int;利用li或者ld输出。
long long:在64位编译器下无区别，在32位编译器下，long为4个字节。long long为8个字节。long long int=long long

说明符号位的

unsigned：无符号，代表当前变量取值只能是正数和0.输出无符号数，用%u。取值范围变大。
signed：有符号，默认就是这个,这个使用普通的取出符号即可。

注意：上面的说明长度和说明符号位的可以混合使用
例如：unsigned long int num=99;
