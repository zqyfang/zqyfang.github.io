  • 抽象
    抽象类，抽象方法都被abstract修饰，抽象方法没有方法体（{}内部的部分）。
    A:有抽象方法的类一定是抽象类或接口
    B：抽象类中不一定有抽象方法。
    C:抽象类不能被实例化。
    构造方法作用：实例化对象，并且初始化成员变量。但是抽象类时候类都是抽象的了，更不可能实例化出对象，所以构造方法只是为了初始化成员变量
    抽象方法必须没有方法体，而且必须abstract修饰。
    public abstract void name();
  • 接口
    interface Animal{}
    接口和抽象类区别：接口全是抽象方法（当然也可以啥也没有），比抽象类更抽象（但是也可以放常量）。
    A:接口不能被实例化
    B:接口中没有变量，只有常量，因为接口中变量都有默认修饰符，public static final。
    C:接口没有构造方法（抽象方法有，因为抽象方法需要初始化变量），但是接口只有静态常量，随类加载而加载。
    D:接口中方法都是抽象的。即使不加abstract也是抽象的，默认有public abstract。
    E:类实现接口时候，可以多实现（不是多重实现），并且可以在继承一个类基础上实现多个接口
    F：接口和接口可以继承，可以多继承。

第三部分

  • 封装一般和set，get方法配合使用。
  • 构造方法:给对象进行初始化。
    如果不显示给构造方法，系统会默认给个无参数构造方法。
    如果一旦给了带参构造，系统不会给出无参数构造。

  • 给成员变量赋值有两种方法
    A：set,get方法
    B：构造方法。

  • this关键字：代表其所在方法所属对象的引用，所以构造方法赋值.

    如果不加this，age还是赋值给age，就近原则还是赋值给的布局变量，而不是成员变量

  • 构造代码块：先于构造方法执行,把所有构造方法中的共同内容定义在构造代码库中。
  • static关键字
    static修饰的成员变量只有一个，不会重复创建，所以也就是说是共用的，这也是其使用场景。
    特点：

A：随类加载而加载
B：优先于对象存在
C:如果该内容被所有对象所共用，就用静态修饰。
D:直接被类名所调用。
注意事项：

A：在静态方法中没有this关键字，因为静态是随着类加载而加载，而this是随对象而存在。
B:静态方法只能访问静态成员变量或静态成员方法。

第四部分

  • 继承：
    子类可以访问父类非私有的属性和行为。
    注意，私有的只有在类内部才能使用，即使是该类本身的对象。

  • 特点

    A：继承java中只能单继承
    解析：因为如果支持多继承，多个父类同时含有同样方法，会照成调用不明确
    B：支持多重继承
    C：num:布局变量，this.num:成员变量，super.num:父类的变量
    this:代表本类对象的引用
    super:代表父类内存空间的引用
    构造方法：
    this.(参数)：本类的构造方法
    suoer.(参数)：父类的构造方法

    成员方法：
    this.方法名（）—本类方法
    super.方法名（）—父类方法

  • 重写
    A：父类的私有方法不可以被重写。
    B:子类访问权限要大于或等于父类的权限
    C:静态的方法只能被静态所重写
    D:看案例：

  • 继承的构造关系：

子类构造默认都去访问父类无参构造方法；
在子类构造方法中第一行，不论写不写，默认都有一行super();
如果父类没有无参构造，（但是有有参构造），子类可以通过super(参数)指定要访问的父类构造。

  • 顺序：静态代码库》》构造代码块》》构造方法

面试题：因为静态代码块和类一起加载
父静态代码块》子静态代码块》父构造代码块》父构造方法》子构造代码块》子构造方法

应用场景：例如本地保存的配置文件，有修改需求时候，例如增加某个节点，此时可以使用序列化对应对象，然后生成json字符串，并保存为文件。

枚举：就是用来表示几种固定类型的取值

枚举的本质就是基本数据类型，整型

枚举的定义

enum 枚举类型名称{
    取值，
};
例子：
//定义枚举类型
enum Gender{
    kGenderMale,//命名规范，最好如此
    kGenderFemale
};
//定义枚举变量
enum Gender sex;
sex=male;
sex=female;

注意点：枚举类型本质是整型，所以枚举类型除了可以接受枚举的固定取值之外
（此时为male和female），还可以接受其他整型的值，也就是说枚举类型可以
当做int类型变量使用。默认枚举的第一个取值就是0，第二个就是1，后面取值递增。

全局变量和局部变量

全局变量(存储在静态区中，随着程序的启动而创建，随着程序结束而结束)

写在函数，代码块，形参列表之外的变量
全局变量没有进行初始化，系统默认将全局变量初始化为0
作用域：从定义的哪一行到文件末尾
extern int num:extern用来声明一个全局变量，但是不会开辟内存空间
也就是说，int num；这种和上面extern声明的全局变量都是外部全局变量，只是有没有默认初
始值的区别，而且即使extern修饰还是能在其他文件修改这个值，因为是同一段内存空间，所以
出现static关键字，这样就是内部全局变量了。

外部全局变量，可以被其他文件访问的全局变量

> 可以定义多个同名的外部全局变量
 多个同名的外部全局变量指向同一块存储空间

内部全局变量，给全局变量加上static关键字，只能被当前文件访问

> 可以定义多个同名的外部全局变量
 多个同名的外部全局变量如果不在同一个文件中，那么指向不同存储空间

局部变量(存储在栈中，作用域结束会自动释放)

局部变量就是定义在函数，代码块和函数形参列表中的变量
相同作用于，不能出现同名的局部变量
局部变量如果没有初始化，那么局部变量是一些随机值
static修饰局部变量：会延长局部变量的生命周期，并且会更改局部变量存储的位置，将局部变量从栈转移到静态区中。
static修饰时候：执行定义布局变量的代码的时候就会分配存储空间，程序结束释放该空间(只是比全局变量稍弱，全局变量。程序启动就分配空间)。

extern和static对函数的作用

函数也分为，内部函数(只能在当前文件访问的函数)和外部函数(函数默认为内部函数)
内部函数：返回值前加上关键字static
static void demo(){};//定义一个内部函数
static void demo();//声明一个内部函数，注意此声明不要放到头文件里面，没有任何意义，因为即使别人引用了头文件，内部函数还是不能使用该函数。
extern关键字：加在函数返回值前，就是外部函数，但是一般不写，因为默认就是外部函数。

预处理指令，把文件翻译成0，1之前的操作，一般都是以#开头

源代码->预处理->汇编->二进制->可执行程序

宏定义(不带分号)，发生在预处理期间

作用：程序翻译成0，1之前，会将所有宏名替换为值

不带参数的宏定义

> #define 宏名 值

注意：一般宏名都大写，多个单词之间用下划线隔开，宏定义不加；
宏定义作用域：从定义哪一行开始到文件末尾。
可提前结束宏定义作用域，在要结束的地方加上#undef COUNT即可。

带参数的宏定义

#define SUM(v1,v2) v1+v2:此时不需要写数据类型
注意：宏定义不会做任何运算，只是做替换而已。
下面直接类似函数使用即可
函数和宏定义：当功能比较简单的时候推荐宏定义，效率更高。
案例：注意结果
#define SUM(v1,v2) v1*v2
int main(int argc,const char *argv[]) {
    printf("%d\n",SUM(5+5,4+4));//输出29
    return 0;
}
解析：因为宏定义只是替换，不进行任何运算，此时真的执行时5+5*4+4结果为29
所以可以更改如下：
#define SUM(v1,v2) (v1)*(v2)
int main(int argc,const char *argv[]) {
    printf("%d\n",SUM(5+5,4+4));//输出80
    return 0;
}
但是当进行多次运算的时候，因为优先级的原因，建议最好把结果也加上圆括号。
#define SUM(v1,v2) （(v1)*(v2)）
int main(int argc,const char *argv[]) {

数组注意点

没有索引为2的元素，此时给arr1赋值为44，则直接赋值给arr1下面的arr数组的
第一个元素，所以arr【0】值为44. 
函数中的形参数组传递：因为数组名称就是数组地址，所以当把数组传递给函数时

数据类型传递的时候，相当于新开辟一段内存，在函数结束的时候，那片内存地址
就会收回，所以这是(值传递)，类似于copy不能修改原来的值，这就是变量形参

原码，补码，反码

数据存储中内存都是存储的二进制，最终存储在内存中的是补码
反码：正数的反码就是正数原码
补码：正数的补码就是正数的原码
所以：正数，三码合一。
案例：通过-9说明负数的存储情况
 -9是int类型，在mac系统上面，占据四个字节，也就是32位。
二进制(原码)第一位是符号位，1代表负数，0代表正数。
 -9的原码就是：10000000 00000000 00000000 00001001
反码为符号位不变，其他位取反
 -9的反码就是：11111111 11111111 11111111 11110110
补码为反码加一
 -9的补码就是：11111111 11111111 11111111 11110111 
总结：之所以有原码，补码，反码的区分，是因为为了方便计算机计算。因为原码最
高位是符号位，如果直接用原码计算，还需要判断是整数还是负数，但是如果用反码，
补码此时已经不区分符号位了，直接计算就行，提高了效率。
案例：1+（-1）=0
00000000 00000000 00000000 00000001  1的补码
11111111 11111111 11111111 11111111  -1的补码
——————————————————————————————————————
00000000 00000000 000000000 00000000 
解析：本来最高位应该是1，但是因为超出四字节，被截取，此时最高位为0为正数，顾结果不需要考虑三码，直接为0.
案例二：-9加1
11111111 11111111 11111111 11110111  -9补码
00000000 00000000 00000000 00000001   1补码
——————————————————————————————————————
11111111 11111111 11111111 11111000   -8补码
解析：最高位为1是负数，先求反码为：11111111 11111111 11111111 11110111
然后求原码为：10000000 0000000 00000000 00001000 此时为-8.

移位运算

左移：<<  左移有可能改变数的正负性，因为左移最高位（符号位）会被抛弃，后面0。
9<<1=9*2(1)=18
右移: >> 右边移动位被抛弃，左边缺失位补充符号位。
9>>1=9/2(1)=4 C语言中整数除以整数等于整数
注意：负数移位(左右)的时候，需要转换为补码在进行移位，移位后，在转化为源码得到值。

int temp=a;
a=b;
b=temp;
方式二：
a=a+b;
b=a-b;
a=a-b;
方式三：
a=a^b;
b=a^b;
a=a^b;

类型说明符号

说明长度的（可用于修改类型所占用的存储空间的大小，一般和int配合使用）

short: short int为两个字节，short int=short。输出使用%hi或者%hd。
long:例如long int num=12411;本来int是四个字节，但是加上之后，为8个字节。long int=int;利用li或者ld输出。
long long:在64位编译器下无区别，在32位编译器下，long为4个字节。long long为8个字节。long long int=long long

说明符号位的

unsigned：无符号，代表当前变量取值只能是正数和0.输出无符号数，用%u。取值范围变大。
signed：有符号，默认就是这个,这个使用普通的取出符号即可。

注意：上面的说明长度和说明符号位的可以混合使用
例如：unsigned long int num=99;
