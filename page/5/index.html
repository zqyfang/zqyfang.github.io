C语言笔记_03(数组相关)

发表于 2017-08-06 | 分类于 C语言

数组：存储同一类型的数据

 定义方式
    //定义方式一：
    int arr[3];
    arr[0]=1;
    arr[1]=2;
    arr[2]=3;
    //定义方式二：
    int arr1[3]={1,2,3};
    //部分初始化：
    int arr2[3]={1,2};
    printf("%i\n",arr2[2]);//部分初始化时候，未初始化的元素，默认值是0
注意：如果没有对数组进行初始化(完全初始化和部分初始化)，则内部可能是垃圾
值，不要使用。另外，数组的元素个数不能使用变量，不然的话数组中元素是一些
随机值。
如果定义的同时进行初始化，元素的个数可以省略

例如：int arr[]={1,2};
案例：给数组指定位置元素赋值
方式一：arr[100]=100
方式二：int arr[100]={[99]=99,[100]=100};
错误案例：不能再定义之后在进行整体初始化，会报错，只能单个赋值。
int arr[3];
arr={1,2,3};//此处是错误的。

遍历数组

数组分类

数值数组  存储数值的
字符数组  存储字符
指针数组  存储指针(地址)
结构数组  存储结构体类型的数据

数组说明

变量内存寻址从大到小，所以存储数据时会从高字节开始存储
数组存储内存寻址从小到大，从所占用字节的低字节开始。
说明案例：
    int arr[3];
    arr[0]=1;
    arr[1]=2;
    arr[2]=3;
    for(int a=0;a<sizeof(arr)/sizeof(arr[0]);a++){
        printf("内存地址%d\n",&arr[a]);
    }
输出：内存地址1606416428，内存地址1606416432，内存地址1606416436
可知数组是从数组所占用字节的低字节开始存储。
注意：数组地址是其所占用存储空间最小的一个地址，即数组的地址就是数
组第一个元素的地址，另外数组名就是数组地址，&arr和arr都是输出数组地址值。
另外，数组中元素存储和变量一模一样，从高字节到低字节存储。

数组注意点

int arr[3]={1,2,3};
int arr1[2]={4,5};
arr1[2]=44;
printf("%d\n",arr[0]);//44
说明：数组本身相当于一个变量，所以这两个数组是从大到小寻址，加入内存类似
于excel表，从上往下分别是从0开始变大，则arr位于arr1的下方，但是此时arr1
没有索引为2的元素，此时给arr1赋值为44，则直接赋值给arr1下面的arr数组的
第一个元素，所以arr【0】值为44. 
函数中的形参数组传递：因为数组名称就是数组地址，所以当把数组传递给函数时
候，传递过去的是地址引用，所以在那个函数中修改了数组的元素的值就是修改的
原来的数组的对应元素的值，因为地址指向同一片内存，此处要和变量区分，基本
数据类型传递的时候，相当于新开辟一段内存，在函数结束的时候，那片内存地址
就会收回，所以这是(值传递)，类似于copy不能修改原来的值，这就是变量形参
和数组形参的区别。

解析：此案例不论数组长度多么的大，都是只输出两个元素，因为数组名称就是地址，
也就是指针，指针在C语言中永远都是占据8个字节，而后面具体int元素，都是四个字节
，所以一直计算的值就是2.
解决：可以外接计算好长度，传递多参数过来解决。

字符串

定义字符串变量
char str[]="qiang";
注意：字符串变量和字符数组有区别，C语言规定，字符串变量必须以\0结束
，所以字符串变量的元素个数比字符数组元素个数大一，多一个\0；
char str[]={'a','b'};//这里是字符数组，不是字符串变量
输出字符串：%s。从传入的地址开始逐个取出，直到遇到\0为止。

案例：
char str[]="lnj";
char str2[]={'x','m','g'};
printf("%s\n",str2);//xmglnj
解析：str和str2作为变量内存寻址是从大到小，所以内存中str2位于str上面，
而%s遇到\0才结束，此时取出完毕str2元素时候，没有遇到\0，会继续取出str
元素直到遇到\0。
其他初始化方式：
char str[]={'l','m'.'j','\0'};//只要显示指出\0此时就是字符串变量
char str[10]={'l','m','j'};
这也是字符串变量，因为未完全初始化的数组，后面默认值是0.而字符\0的ascii对应值就是0.

输出字符串

方式一：通过printf（“%s”,str）;输出字符串
缺点：如果想换行，必须加上\n。
优点：可以自定义字符串输出格式
方式二：puts()
优点：会自动换行
缺点：无法自定义格式，只能原样输出

输入字符串

方式一：

scanf("%s",buf);
注意点：会以空格，tab，enter作为接收符号。
方式二：gets()
说明：通过此方法输入系统会报警告，因为可能超出范围。

gets(buf);
优点：可以利用gets接收字符串，并且可以在字符串中输入空格，tab。

获取字符串长度：首先引入头文件<string.h>，然后使用strlen()方法获取字符串长度(不包括\0)
    char buf[]="qiangge";
    size_t length=strlen(buf);
    for (int a=0; a<length; a++) {
        printf("%c\n",buf[a]);
    }

strcat（dest,src）：字符串连接,将src数据拼接到dest后面
注意：要想使用拼接函数，dest必须是数组，而且数组的长度必须大于拼接之后的长度。
strncat(char*,const char*,size_t):参数一二和上面方法参数意思一样，参数三，指的是需要拼接的个数。
例如：size为1的时候，只会把参数二的第一个字符拼接到参数一中。
strcpy: 字符串拷贝。
    char str1[10]="lnjads";
    char str2[]="cool";
    strcpy(str1, str2);//会把str2的内容拷贝并覆盖到str1.
    printf("%s\n",str1);//cool
注意点：目标容积必须能够存放拷贝的数据。
strncpy(char*,const char*,size_t):参数三，需要拷贝几个字符。
注意：此处比如说只拷贝一个字符，则只会逐个替换（结果为cnjads）.
strcmp（str1,str2）: 字符串比较,返回值为整型的值(值=0则字符串相等，<0则str1<str2).
原理：取出每一个字符，进行逐个比较，如果发现不相等就不会再继续往下比较了。
例如：“abc”和"acc"比较，逐个取出字符，a和a比较相等，b和c比较已经不相等，返回负数，后面的c已经不比较了。

函数的声明和实现

C语言中默认在调用之前，声明定义函数，如果不定义函数，一定要在调用之前声明函数。
#include <stdio.h>
void showHello();//函数声明
int main() {
    showHello();
    return 0;
}
void showHello(){//函数定义
    printf("hello");
}
注意：如果函数返回值是int类型，那么不写声明，也不会报错。而且函数声明可以省略形参，
函数声明只需要在函数被使用之前就可以了，它可以写在函数的外面，也可以写在函数里面。
C语言中函数定义不能重复，但是函数声明可以重复，因为函数声明仅仅是为了告诉系统有这个函数，
接受什么，返回什么，所以多告诉几次也无所谓。

argc:系统传递进来的参数个数
argv:系统传递进来参数的实际值

原码，补码，反码

数据存储中内存都是存储的二进制，最终存储在内存中的是补码
反码：正数的反码就是正数原码
补码：正数的补码就是正数的原码
所以：正数，三码合一。
案例：通过-9说明负数的存储情况
 -9是int类型，在mac系统上面，占据四个字节，也就是32位。
二进制(原码)第一位是符号位，1代表负数，0代表正数。
 -9的原码就是：10000000 00000000 00000000 00001001
反码为符号位不变，其他位取反
 -9的反码就是：11111111 11111111 11111111 11110110
补码为反码加一
 -9的补码就是：11111111 11111111 11111111 11110111 
总结：之所以有原码，补码，反码的区分，是因为为了方便计算机计算。因为原码最
高位是符号位，如果直接用原码计算，还需要判断是整数还是负数，但是如果用反码，
补码此时已经不区分符号位了，直接计算就行，提高了效率。
案例：1+（-1）=0
00000000 00000000 00000000 00000001  1的补码
11111111 11111111 11111111 11111111  -1的补码
——————————————————————————————————————
00000000 00000000 000000000 00000000 
解析：本来最高位应该是1，但是因为超出四字节，被截取，此时最高位为0为正数，顾结果不需要考虑三码，直接为0.
案例二：-9加1
11111111 11111111 11111111 11110111  -9补码
00000000 00000000 00000000 00000001   1补码
——————————————————————————————————————
11111111 11111111 11111111 11111000   -8补码
解析：最高位为1是负数，先求反码为：11111111 11111111 11111111 11110111
然后求原码为：10000000 0000000 00000000 00001000 此时为-8.

移位运算

左移：<<  左移有可能改变数的正负性，因为左移最高位（符号位）会被抛弃，后面0。
9<<1=9*2(1)=18

9>>1=9/2(1)=4 C语言中整数除以整数等于整数
注意：负数移位(左右)的时候，需要转换为补码在进行移位，移位后，在转化为源码得到值。

说明长度的（可用于修改类型所占用的存储空间的大小，一般和int配合使用）

CSS定位和浮动

发表于 2017-08-06 | 分类于 前端

css定位和浮动（清除）

css定位

定位概念：定位可以有负数的概念，相关属性，left，right，bottom，top。

  1. 相对定位：不脱标，原来位置其实还占据着，只是形影移动罢了。
  2. 绝对定位：脱标，不需要display:block;就可以设置宽高。
  3. 固定定位：就是固定而言的。
  4. 绝对定位分析：margin：0，auto可以使盒子居中，但是只是针对处于标准流中而且有明确宽度的盒子而言的。
    当绝对定位时候，已经脱标，这样是无法使盒子居中的，但是可以通过left:50%; margin-left:负的宽度的一半。

总结：一般子绝父相。（后面描述针对所有定位有效）而且bottom是相对页面的左下角来说,而不是可见区域（屏幕）左下角。

乱序知识点：要想设置百分比宽高，其父元素必须有宽高，设置页面整体时候，必须先把html，body设置百分百。

浮动和清除

浮动的概念：浮动的元素会脱离标准文档流，向左或者向右浮动，直到遇到父元素或者另一个浮动的元素.

  1. 浮动的元素可以设置宽高并且内联排列，是介于inline和block之间的存在，和浮动类似的有inline-block.
  2. 浮动的元素脱离标准文档流，并且可能会覆盖标准流元素。

  3. 浮动会导致父元素高度坍塌.

    解析：浮动的元素脱离标准文档流，并不占据文档流位置，自然无法撑开父元素高度。

清除浮动：是通过在别的元素上清除浮动来撑开高度，而不是在浮动的元素上面清除。

  • 方式一：

clear：both；margin属性失效,而且无法给父元素撑出高度。
例如：两个div都浮动，如果不加clear：both属性，下一个div也会贴过来，加clear：both（在第二个加即可）
两个div会分行排列，但是会第一个div下部和第二个div上部贴合，margin属性也无效。

  • 方式二：

通过overflow: hidden;给父元素撑出高度，注意此属性用于父元素上面。

  • 方式三：

内墙法，外墙法等。也是通过clear，但是弥补了属性失效的缺点
例如：此时是随意新建一个div不需要给高度，只需要设置clear属性即可，注意区分该方式和方式一。

  • 方式四：给父元素设置高，不推荐。

总结：清除浮动有四种方式，但是设置高太过于死板，方式一无法给父元素撑出高度，而且margin属性失效。
推荐使用方式二三。

inline-block和浮动的区别

inline-block：既可以设置宽高，又可以内联排列，介于inline和block之间的效果。
内联排列时候，例如标题栏的排布（ul）,推荐使用inline-block。
但是：不浮动则不脱标，此时li之间有空格符号、这个空格符将会产生4px的边距。故而使用inline-block时候
需要注意空格符号造成的边距问题。
属性：当条目高度不一的时候，可以通过vertical-align: top;让元素顶部对齐，同理也可以底部对齐。
